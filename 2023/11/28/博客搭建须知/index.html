博客搭建须知

发表于2023-11-28|更新于2024-12-05
|总字数:501|阅读时长:1分钟|浏览量:

正文
Hexo配置博客:
_config.yml需要修改的内容
url: https://tangtiaodaoaishang(github用户名).github.io
deploy:
type: git
repository: git@github.com:tangtiaodaoaishang(github用户名)/tangtiaodaoaishang(github用户名).github.io.git
branch: main

一 运行个人博客页面:
1.在个人博客路径下右键Git Bash Here,打开git命令行
2.输入hexo d运行
3.访问github域名:https://用户名.github.io(其中我的github用户名为tangtiaodaoaishang)即可访问页面
–我的个人博客地址:https://tangtiaodaoaishang.github.io

二 在个人博客发布文章:
1.在个人博客路径下右键Git Bash Here,打开git命令行
2.输入hexo new “My New Post”执行（hexo new “自定义文章名”)
3.然后博客目录下source 文件夹里的_post文件夹中会出现一个 My New Post.md 文件
4.以后每次发布文章都是这两条命令:
hexo clean # 刷新缓存
hexo g # 生成页面
hexo d # 部署发布

三 更换个人博客主题(我的主题:next) – git clone https://github.com/theme-next/hexo-theme-next themes/next
–主题页面内容在source目录下
1.在Themes | Hexo官网选择主题背景
2.进入博客目录打开 Git Bash Here 下载主题(根据文档配置下载)： 例如git clone https://github.com/theme-next/hexo-theme-next themes/next
3.修改_config.yml中的theme属性值,更换为新主题名
4.依次执行hexo clean hexo g hexo s三个命令即可

四 常用命令
hexo new “name” # 新建文章
hexo new page “name” # 新建页面
hexo g # 生成页面
hexo d # 部署
hexo g -d # 生成页面并部署
hexo s # 本地预览
hexo clean # 清除缓存和已生成的静态文件
hexo help # 帮助

五 修改已发布的文章内容
1.在source目录下的_posts目录中找到对应文章,使用编辑器打开并修改内容
2.终端依次执行hexo c,hexo g,hexo d三个命令
3.执行hexo s运行页面
